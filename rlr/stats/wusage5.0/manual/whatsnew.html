What's new in wusage 5.0 patchlevel 6?

Wusage 5.0 patchlevel 6 corrects a bug which caused bits/sec and bytes/sec fields in the daily report to be off by a factor of exactly 24. (They were being calculated as if a day was one hour long.)

What's new in wusage 5.0 patchlevel 5?

Wusage 5.0 patchlevel 5 (known as version 5.02, under Windows 95/NT) corrects the following problems with the original 5.0 release:
  • Using custom HTML no longer crashes the program.
  • The nobytes option now works correctly.

What's new in wusage 5.0?

Version 5.0 adds a host of new features not found in version 4.0. Here is a list of the major new features and corrections in the 5.0 series.

Important: some of these new features won't "kick in" unless you turn them on by setting options in your configuration file and configure your server properly, so be sure to take a look at this list.

  • nozeroes option for totals
  • User agent (web browser) report
  • Information on DNS problems with win95 and NT
  • Referrer (incoming link) report
  • Frequently followed trails report
  • User visits report
  • Referrers can be included in visits
  • Easy subdomain reports for intranets
  • Reports are now generated for the "current" day, week or month in progress
  • Reports are now displayed in the order configured
  • All reports can now be individually enabled or disabled
  • All pie charts can be turned off
  • The -b, -e and -s options now work together
  • Reports can be offset by any number of hours
  • Query strings after ? can be optionally kept
  • Better formatting of all numeric output
  • Index page now includes total, average and latest accesses
  • Requests which generate errors are no longer displayed as regular document requests
  • Some support for incorrect date formats in log files
  • File extension for HTML documents is configurable
  • "Fast" DNS option looks up names in reports only
  • Control of bar chart and pie chart colors
  • 3D bar charts
  • Error log entries properly ignored (no more "To" and "Failed" in the documents report)
  • Several patterns can be compared in the same 3D bar chart
  • Most HTML tags can be changed
  • All English text output can be changed
  • Referrers are included in "documents not found" report
  • Bytes can be suppressed from reports
  • Several ways to recognize the virtual domain name in log files
  • Dates substantially in the future are ignored (works around a common Apache problem)
  • Accesses by day report, much like the accesses by hour report
