If present, the domains option indicates that wusage should output information about the Internet domains which accessed the server. If the domains option does not appear or is set to off, no domain-related information will be included in reports. If your access log does not contain full host names for most sites, the domains option will not produce interesting reports unless the dns option is set to on, which can be a very slow operation. Domain reporting is useful to determine which Internet domains, such as com (commercial, usually US), edu (educational, usually US), and ca (Canada), accessed the server most often. Using the domaingroups option, you can also combine accesses from several domains under one name. If your server is exclusively accessed from within an Intranet, be sure to check out the subdomainsof option, which is new in version 5.0. This information often proves to be valuable demographic data about your users. See also topdomains, fakedns, domaingroups, subdomainsof, and subdomains.
