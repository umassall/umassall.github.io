logfile

logfile         /path/to/log/file/access_log
logfile indicates the location of the access log file created by your web server. This file must exist, and must be readable by wusage. See your web server's documentation and configuration files for more information about the location of this file. The logfile option can be overridden by the -l command line option. The special filename - indicates that wusage should expect to find access log data at standard input, which is useful in a Unix or NT "pipeline." See the logdir option and the section on reading from many log files for information about working with more than one log file.
