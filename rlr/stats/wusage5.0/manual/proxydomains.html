proxydomains

 
proxydomains on
proxydomains off
proxydomains nopie
If present, the proxydomains option indicates that wusage should output information about the Internet domains which were accessed through your proxy server. This option is only useful if you are analyzing the logs of a proxy server. If the proxydomains option does not appear or is set to off, no proxy domain-related information will be included in reports. Domain reporting is useful to determine which Internet domains, such as com (commercial, usually US), edu (educational, usually US), and ca (Canada), were accessed through the server most often. Using the domaingroups option, you can also combine accesses from several proxydomains under one name. If your proxy server is exclusively used to access sites within an Intranet, be sure to check out the subdomainsof option, which is new in version 5.0. See also topproxydomains, fakedns, domaingroups, subdomainsof, and subdomains.
