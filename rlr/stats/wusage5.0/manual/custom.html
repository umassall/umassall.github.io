Controlling the Order of Reports

Wusage 5.0, unlike previous versions, outputs reports in the order they are turned on in your log file. Reports that are not explicitly turned on, but which are turned on by default, appear at the top or bottom of the report. If you do not turn on these reports explicitly, they appear in the following order. All other reports are turned off unless you explicitly turn them on.

Inserted at the top of the report, before any explicitly configured reports:

unique
hours
days
documents

Inserted at the end of the report, after any explicitly configured reports:

trails visits resultcodes

Output Customization Options

The output of Wusage 5.0 can be customized to a substantial degree. In addition to the old header and footer options, every piece of English text output by Wusage can be changed in the configuration file. The user also has a substantial degree of control over the HTML tags output by version 5.0. It is by no means necessary to use these options, but users who want this sort of power have a tremendous amount of control with version 5.0.

HTML and Text Output Options

These options control every piece of English text and nearly every HTML tag output by Wusage.

The links in the alphabetical list below jump to appropriate points in an example configuration file containing all of these options, with their default settings.

You do not need to set all of these options in your own configuration file. Just insert the options you wish to change from the default settings.

If you would prefer to work with a text file that already contains these options in a form suitable for inclusion in your configuration file, see the file customconf.txt.

Improper use can, of course, result in pages that do not render correctly in your web browser. When in doubt, just change the text, and leave the HTML tags (between < and >) alone. Also note that %s and %d are sometimes used as placeholders for filenames, numbers, and so forth that are inserted by the program. For best results, use the default settings as a starting point for your own variations.
